# 05_DECISIONS

## 2026-05-31

- アプリ名は「トレログ」とする。
- α版はHTML/CSS/JavaScriptで作る。
- React、TypeScript、バックエンドはまだ使わない。
- 保存はlocalStorageを使う。
- LLM APIはα版では使わない。
- カメラ姿勢推定は将来機能とする。
- スマホアプリ化は将来機能とする。
- まずはブラウザで動くWebアプリとして完成させる。
- インタラクト要素として、α版では音楽アップロードと再生を入れる。
- トレーナーは初期版では固定画像と定型コメントで表現する。
- 記録の保存単位は、複数メニューをまとめた1日分ではなく、1回の保存につき1つの種目ログとする。履歴やEXP計算を種目単位で扱いやすくし、1日に複数ログを残せるようにするため。
- 日ごとの達成判定は、その日に1件以上の種目ログがあれば達成扱いとする。
- チェックボックス式メニューは保存対象ではなく、今日のクエスト候補または達成済み表示として残す方針だったが、2026-06-01の整理で廃止する。

## 2026-06-01

- 「今日のクエスト候補」チェックボックス欄は廃止し、「今日の目標」カードに置き換える。種目選択は種目ログ入力のプルダウンに一本化し、画面上の役割重複を避けるため。
- 種目プルダウンには「その他 / 新規追加」を追加し、自由種目名を入力できるようにする。
- 筋トレ開始ボタンはタイマー操作を兼ねる。状態は開始前、実行中、一時停止中の3状態とし、ボタン文言を「筋トレ開始」「一時停止」「再開」で切り替える。
- 今回は本格的なEXP計算ではなく、今日の目標カード用の簡易スコアのみ実装する。
- 保存後はタイマーをリセットし、2秒間保存ボタンを無効化して重複保存を防ぐ。
- 種目ログはlocalStorageの `trelog_records` に保存し、ページ再読み込み後も履歴と集計を復元する。
- 今回は簡易スコアをそのままEXPとして保存し、累計EXPとレベルを実データから反映する。
- 今日の目標スコアは固定で100ptとし、目標到達目安は現在の入力条件から必要量を逆算して表示する。
- 継続日数は休憩チケットを使わず、記録が存在する連続日だけを数える方針だったが、追加実装で休憩日も継続扱いに含める。

## 2026-06-01 追加

- アプリ状態はlocalStorageの `trelog_state` に保存する。休憩チケット、休憩日、目標達成報酬の受け取り済み日付を記録するため。
- 起動時に昨日以前の未記録日で継続が途切れそうな場合、休憩チケットを自動消費して `restDates` に追加する。今日の未記録は当日中の挽回余地があるため休憩日にしない。
- 継続日数は記録日と休憩日の両方を継続扱いにして計算する。
- 今日のスコアが100pt以上になったとき、同日に未受け取りなら達成済み日付を `claimedGoalRewardDates` に保存して重複処理を防ぐ。休憩チケットは付与しない。
- トレーナーコメントは開始、一時停止、再開、保存、目標達成、休憩チケット自動使用、入力エラーで分岐させる。

## 2026-06-01 報酬仕様調整

- 画面表示上の「凍結チケット」は「休憩チケット」に変更する。内部状態も `restTickets`、`restDates` に整理し、旧 `freezeTickets`、`frozenDates` は読み込み時に移行する。
- 今日の目標達成では休憩チケットを付与しない。達成コメントと達成済み状態の保存だけを行う。
- 休憩チケット付与はレベル報酬へ移し、Lv.5、Lv.10、Lv.15のように5レベルごとに1枚付与する。
- レベル報酬の二重受け取り防止のため、`trelog_state.claimedLevelRewards` に受け取り済みレベルを保存する。
- ダッシュボードにレベル進捗バーを追加し、現在レベル内のEXP進捗と次レベルまでの必要EXPを表示する。
- 今週ランクは月曜始まりの記録日数と週間スコアからS〜Dで算出する。
- 開発用デバッグパネルは `DEBUG_MODE` で表示を切り替え、localStorageの確認とテストデータ操作に使う。

## 2026-06-01 PWA土台

- α版をブラウザで動くWebアプリとして保ちつつ、ホーム画面追加やスタンドアロン表示へ進められるようにPWA化の土台を追加する。
- 今回はmanifestとservice workerの最小構成に留め、Push通知、スマホウィジェット、クラウド保存、ネイティブアプリ化は実装しない。
- PNGアイコンは正式制作前のためmanifestから外し、存在する仮SVGアイコンだけを参照する。存在しないファイル参照によるManifest警告を避けるため。

## 2026-06-01 GitHub Pages向け配置

- GitHub Pagesで公開しやすくするため、Webアプリ本体は `src/` 配下ではなくプロジェクトルートに置く。
- `index.html`、`style.css`、`app.js`、`manifest.json`、`service-worker.js` はルート配置とし、各参照パスはルートからの相対パスで統一する。
- localStorageのキーや保存形式は変更しない。

## 2026-06-01 セッション画面

- スマホ実機では音楽アップロードが主要インタラクトとして使いにくいため、音楽アップロードを主軸にしない。
- 筋トレ開始後の専用セッション画面を主要なインタラクト要素にする。筋トレ中にトレーナーコメント、タイマー、現在種目、目標進捗を見やすくするため。
- 音楽再生機能は既存のまま残すが、複数音源、プレイリスト、ループ再生は将来タスクとする。
- 種目ログの保存形式、EXP計算、localStorageキーは変更しない。

## 2026-06-01 セッション入力改善

- セッション画面だけで筋トレの実行、入力、保存を完結しやすくするため、回数式の入力欄をセッション内に追加する。
- セッション中の目標到達目安は、保存済みスコアだけでなくセッション中の見込みスコアも反映する。
- 時間式はセッション経過時間を保存値とし、回数式はセッション内の回数入力値を保存値とする。
- セッション内にも音楽ファイル名、音楽ファイル選択、再生 / 停止を置くが、複数音源、プレイリスト、ループ再生はまだ実装しない。

## 2026-06-01 リザルト画面

- 種目ログ保存後の達成感を強めるため、保存成功後にリザルトオーバーレイを表示する。
- リザルト画面は表示専用とし、記録保存、EXP計算、レベル報酬、休憩チケット付与の既存処理は変更しない。
- 今日の目標達成、レベルアップ、休憩チケット報酬は保存処理の結果から表示する。報酬の二重付与は既存の受け取り済み状態で防ぐ。
- 祝福演出はCSSアニメーションに留め、デザイン全面改修は行わない。

## 2026-06-01 リザルト表示改善

- リザルト画面は、結果を一目で追えるように「大見出し、トレーナー、今回の記録、獲得報酬、今日の進捗、成長」の順に整理する。
- リザルト時のトレーナーコメントは、通常記録、今日の目標達成、レベルアップ、休憩チケット獲得、低きつさ、高きつさで分岐する。
- 演出は状態に応じて変え、通常記録は控えめなキラキラ、今日の目標達成とレベルアップは紙吹雪にする。
- 今日の目標達成、Lv.UP、休憩チケット獲得は、該当時だけバッジ風に表示する。

## 2026-06-01 セッション終了とリザルト演出改善

- 時間式セッションでは、一時停止を必須にせず、`running` 中でも「記録して終了」からその時点の経過時間を保存できるようにする。スマホ実機で終了操作を1ステップにするため。
- 一時停止は休憩操作として残し、保存時は経過秒数を残したままタイマーだけ止めてから既存保存処理に渡す。
- リザルト演出は画面上部だけで終わらないように、リザルトオーバーレイ全体を覆う演出レイヤーで表示する。
- 通常記録は控えめなキラキラ、今日の目標達成・レベルアップ・休憩チケット獲得は2〜3秒程度の紙吹雪にする。演出はCSSだけで実装し、重い処理は入れない。
