# 2026-06-10 トレーナー選択 v1 画像/休憩仕上げ

- プリセットトレーナー画像は `assets/trainer/public/presets/{trainerId}/` に配置する。
- 各トレーナーは `default`、`cheer`、`result`、`rest` の4差分画像を持てる。
- 画像が不足する場合は同トレーナーの `default`、既存public画像、仮表示の順にフォールバックする。
- 一時停止/休憩/休憩チケット系メッセージもトレーナー別コメントとして扱う。
- IndexedDBにカスタム画像がある場合は、画像はカスタム画像を優先する。ただし、トレーナー選択によって名前、説明、口調、セリフは変わる。
- リザルトEXPゲージは進捗色を赤、ピンク、オレンジ系の暖色に固定し、背景バーとの色差を明確にする。

## 2026-06-09 ホーム画面のモチベーション体験

## 2026-06-09 継続管理とカレンダー表示

## 2026-06-01 時間式ログの秒単位化

- 30秒のプランクなど1分未満の運動も記録できるように、時間式ログは `elapsedSeconds` を保存・表示・集計の基準にする。
- 時間式は経過秒数が1秒以上なら保存可能とし、0秒の場合だけ保存不可とする。
- `value` は互換・表示用の分換算値として残し、`unit` は従来通り `分` のままにする。
- 時間式のスコア計算は `elapsedSeconds / 60` を使い、既存のEXP、レベル、今日のスコア、今週ランク、継続日数の流れは維持する。
- 履歴集約では時間式ログの `value` ではなく `elapsedSeconds` を合計し、60秒未満は秒、60秒以上は分で表示する。

- カスタム画像はIndexedDBに保存し、GitHubには上げない。
- 旧1枚設定の `customTrainerImage` は `custom_default` として互換利用する。
- 応援・祝福・休憩のカスタム画像がない場合は `custom_default` をフォールバックにする。
- 画像本体を含むバックアップJSONはサイズが大きくなるが、サイトデータ削除時の復元に有効なためversion 2でData URLを含める。

## 2026-06-01 バックアップ種別

- バックアップは軽量版と完全版を分ける。
- 軽量版は画像本体を含めず、記録・状態・スコア設定・画像メタ情報だけを含める。
- 完全版は画像本体をData URLとして含むためサイズが大きくなる。
- 通常利用では軽量版、端末移行や完全復元では完全版を使う。
# 2026-06-10 追記: カウンセリング機能 v1

# 2026-06-10 追記: リザルトEXP演出

# 2026-06-10 トレーナー選択 v1 仕上げ

- プリセットトレーナーごとに、ホーム・セッション・リザルト・休憩/継続系の主要コメントを持たせる。
- カスタム画像が設定されている場合は画像を優先するが、トレーナー選択によって名前・口調・セリフは変わる。
- カスタム画像優先時はユーザーにその旨をUI上で説明する。
- EXPゲージは背景と進捗の色差を明確にし、白塗りに見えないようにする。
- プリセット専用画像が未設定でも、既存public画像とCSS仮表示へのフォールバックで表示崩れを避ける。
- カスタム画像優先は現時点では独立した設定値を持たず、IndexedDBに保存されたカスタム画像の有無で決める。バックアップ/復元は既存の `trelog_state.selectedTrainerId` と `trainerImages` を継続利用する。

# 2026-06-10 設定画面アコーディオン化

- 設定画面は機能追加により項目が増えたため、カテゴリ別のアコーディオン/プルダウン形式で整理する。
- デバッグや開発者向け設定は初期状態で閉じ、通常利用の邪魔にならないようにする。
- 設定画面の開閉状態は可能ならlocalStorageに保存するが、アプリデータのバックアップ対象には含めない。
- 設定画面の整理は見た目と導線改善を目的とし、既存の設定データ構造や記録データは変更しない。
