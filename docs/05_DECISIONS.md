# 05_DECISIONS

## 2026-05-31

- アプリ名は「トレログ」とする。
- α版はHTML/CSS/JavaScriptで作る。
- React、TypeScript、バックエンドはまだ使わない。
- 保存はlocalStorageを使う。
- LLM APIはα版では使わない。
- カメラ姿勢推定は将来機能とする。
- スマホアプリ化は将来機能とする。
- まずはブラウザで動くWebアプリとして完成させる。
- インタラクト要素として、α版では音楽アップロードと再生を入れる。
- トレーナーは初期版では固定画像と定型コメントで表現する。
- 記録の保存単位は、複数メニューをまとめた1日分ではなく、1回の保存につき1つの種目ログとする。履歴やEXP計算を種目単位で扱いやすくし、1日に複数ログを残せるようにするため。
- 日ごとの達成判定は、その日に1件以上の種目ログがあれば達成扱いとする。
- チェックボックス式メニューは保存対象ではなく、今日のクエスト候補または達成済み表示として残す方針だったが、2026-06-01の整理で廃止する。

## 2026-06-01

- 「今日のクエスト候補」チェックボックス欄は廃止し、「今日の目標」カードに置き換える。種目選択は種目ログ入力のプルダウンに一本化し、画面上の役割重複を避けるため。
- 種目プルダウンには「その他 / 新規追加」を追加し、自由種目名を入力できるようにする。
- 筋トレ開始ボタンはタイマー操作を兼ねる。状態は開始前、実行中、一時停止中の3状態とし、ボタン文言を「筋トレ開始」「一時停止」「再開」で切り替える。
- 今回は本格的なEXP計算ではなく、今日の目標カード用の簡易スコアのみ実装する。
- 保存後はタイマーをリセットし、2秒間保存ボタンを無効化して重複保存を防ぐ。
- 種目ログはlocalStorageの `trelog_records` に保存し、ページ再読み込み後も履歴と集計を復元する。
- 今回は簡易スコアをそのままEXPとして保存し、累計EXPとレベルを実データから反映する。
- 今日の目標スコアは固定で100ptとし、目標到達目安は現在の入力条件から必要量を逆算して表示する。
- 継続日数は凍結チケットを使わず、記録が存在する連続日だけを数える方針だったが、追加実装で凍結日も継続扱いに含める。

## 2026-06-01 追加

- アプリ状態はlocalStorageの `trelog_state` に保存する。凍結チケット、凍結日、目標達成報酬の受け取り済み日付を記録するため。
- 起動時に昨日以前の未記録日で継続が途切れそうな場合、凍結チケットを自動消費して `frozenDates` に追加する。今日の未記録は当日中の挽回余地があるため凍結しない。
- 継続日数は記録日と凍結日の両方を継続扱いにして計算する。
- 今日のスコアが100pt以上になったとき、同日に未受け取りなら凍結チケットを1枚付与する。報酬済み日付は `claimedGoalRewardDates` に保存して重複受け取りを防ぐ。
- トレーナーコメントは開始、一時停止、再開、保存、目標達成、凍結自動使用、入力エラーで分岐させる。
