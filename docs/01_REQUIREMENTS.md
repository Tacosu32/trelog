# 01_REQUIREMENTS

## α版の機能要件

### 0. 画面構成

- 画面はホーム、記録、履歴、設定の4画面に分ける
- 1つのHTML内で各画面セクションの表示・非表示を切り替える
- 画面下部に固定のナビゲーションを表示し、ホーム、記録、履歴、設定を切り替えられるようにする
- 現在選択中のタブが分かる見た目にする
- メインコンテンツが下部ナビゲーションに隠れないように余白を確保する
- localStorageの保存形式は変更しない

### 1. 日付・記録

- 今日の日付を表示する
- 1回の保存は、1つの種目ログとして扱う
- 1日に複数の種目ログを保存できる
- 日ごとの達成判定は、その日に1件以上ログがあれば達成扱いとする
- 過去の履歴を表示する
- 保存ボタンを押すと、その時点で1つの種目ログとして履歴に追加する
- 保存後はタイマーをリセットし、開始前の状態に戻す
- 種目ログはlocalStorageの `trelog_records` に保存する
- 起動時にlocalStorageから記録を読み込み、履歴表示を復元する

### 2. 筋トレ入力

- 種目を1つ選択して記録できる
- 種目プルダウンには、腕立て、腹筋、スクワット、プランク、ストレッチ、その他 / 新規追加を表示する
- 「その他 / 新規追加」を選んだ場合、新しい種目名を入力できる
- 記録方式は「時間」または「回数」から選択できる
- 種目ごとに対応する記録方式を定義する
- プランク、ストレッチは時間式のみ選択できる
- 非対応の記録方式は選択できない見た目にし、選択中だった場合は自動で対応方式へ切り替える
- 記録方式の補足として「この種目は時間で記録します」のような説明を表示する
- 時間式の場合は、タイマーの経過秒数を `elapsedSeconds` として保存し、表示時に秒または分へ変換する
- 時間式の場合、経過秒数が1秒以上なら保存できる
- 回数式の場合は回数を入力できる
- 回数式の場合、タイマーの経過時間は補助情報として履歴に表示する
- きつさを1〜5で選択できる
- メモを任意で入力できる
- 「今日のクエスト候補」チェックボックス欄は廃止する

### 3. 今日の目標

- ホーム画面に今日の目標カードを表示する
- 今日の目標カードを表示する
- 今日の目標カードには、今日の目標スコア、現在スコア、達成率、おすすめ目安を表示する
- 今日の目標スコアは `DEFAULT_SCORING_CONFIG.dailyGoalScore` を基本値とする
- 開発者用スコア設定がある場合は、その目標スコアを優先する
- 今日の日付のログだけを集計して、今日の現在スコアを表示する
- 現在の種目、記録方式、きつさ、現在スコア、今日の目標スコアに応じて、今日の目標到達までの目安を自動更新する
- 目標到達までの必要量は、残りスコア ÷ 種目係数 ÷ きつさ補正で計算し、小数点が出る場合は切り上げる
- 今日のスコアが100pt以上になったら今日の目標達成とする
- 今日の目標達成時、その日付を達成済みとして保存し、達成コメントを表示する
- 今日の目標達成では休憩チケットを付与しない
- 同じ日に今日の目標達成報酬を複数回受け取ることはできない
- セッション中は、今日の保存済みスコアにセッション中の見込みスコアを足した見込みスコアを表示する
- セッション中の目標到達目安は、見込みスコアを反映して動的に更新する
- 見込みで目標達成できる場合は、記録すると目標達成になることを表示する

### 4. タイマー

- 筋トレ開始ボタンを押すとタイマーが開始する
- 状態は `idle`、`running`、`paused` の3つとする
- `idle` は開始前、`running` は筋トレ中、`paused` は一時停止中を表す
- 筋トレ開始ボタンを押すと `running` になり、セッションオーバーレイを表示する
- `running` 中はセッション画面のボタン表示を「一時停止」に変更する
- 一時停止中はセッション画面のボタン表示を「再開」に変更する
- 保存後はタイマーをリセットする

### 5. 音楽アップロード

- ユーザーが音楽ファイルを選択できる
- 選択した音楽を再生できる
- 音楽再生と筋トレ開始を同じ画面で行える
- セッション画面内にも選択中の音楽ファイル名を表示する
- セッション画面内でも音楽ファイル選択と再生 / 停止ができる
- スマホでは音楽アップロードを主軸にせず、筋トレ開始後のセッション画面を主要なインタラクト要素とする
- α版では音楽プレイリスト、複数音源、ループ再生は実装しない

### 6. 評価

- 種目、記録方式、入力値、きつさから簡易スコアを計算する
- スコア計算用の目標スコア、きつさ倍率、評価プロファイル倍率、種目係数は `DEFAULT_SCORING_CONFIG` にまとめる
- `trelog_dev_scoring_config` がlocalStorageに存在する場合は、開発者用の一時上書きとして `DEFAULT_SCORING_CONFIG` より優先する
- `allowedModes` に含まれない記録方式では開始・保存できない
- 設定画面で評価レベルを選択できる
- 評価レベルは、初心者、標準、慣れている、高負荷向けの4種類とする
- 初期値は標準とする
- スコア計算では、種目係数、きつさ補正の後に評価レベル補正を適用する
- 評価レベル補正は今後保存するログに反映し、過去ログは再計算しない
- 今回は簡易スコアをEXPとして保存する
- 累計EXPを表示する
- 全ログのEXPを合計して累計EXPを表示する
- `level = Math.floor(totalExp / 100) + 1` でレベルを表示する
- 現在レベル内のEXP進捗バーを表示する
- 現在レベル、現在EXP、次のレベルまであと何EXPかを表示する
- 5レベルごとに休憩チケットを1枚付与する
- レベル報酬は `claimedLevelRewards` で受け取り済みレベルを管理し、二重受け取りを防ぐ
- 今日の評価コメントを表示する

### 6.1 今週ランク

- 今週ランクを表示する
- 今週は月曜始まりとする
- 今週の記録日数と週間スコアからランクを算出する
- S: 今週5日以上記録、または週間スコア500以上
- A: 今週4日以上記録、または週間スコア350以上
- B: 今週3日以上記録、または週間スコア200以上
- C: 今週1〜2日記録
- D: 今週記録なし
- 可能なら、今週の記録日数と週間スコアも小さく表示する

### 7. 継続

- 連続記録日数を計算する
- その日に1件以上ログがあれば達成日とする
- restDatesに含まれる日を休憩日とする
- 継続日数計算では、記録日と休憩日の両方を継続扱いにする
- 今日に記録または休憩日があれば今日から数える
- 今日に記録がなく、昨日が記録日または休憩日なら昨日から数える
- 連続している日付だけをカウントし、未記録かつ未休憩の日があればそこで終了する
- 起動時に、昨日以前の未記録日が継続を途切れさせる場合、休憩チケットを自動消費して継続を維持する
- 今日に記録がない場合、今日はまだ休憩日にしない
- 複数日空いている場合は、チケットがある分だけ過去の日付を休憩日にしてよい
- チケットが足りない場合は、それ以上は休憩日にしない
- 休憩チケットの枚数を表示する

### 8. トレーナー

- トレーナー画像を表示する
- 状況に応じた定型コメントを表示する
- α版では会話機能は実装しない

### 9. 履歴

- 履歴画面に集約履歴を表示する
- `trelog_records` の保存形式は変更せず、表示時に履歴を集計する
- 履歴は年、月、日、種目単位の階層で表示する
- 日付行には、日付、その日の合計スコア、目標スコア、目標達成済みかどうか、その日の記録種目数を表示する
- 種目行には、種目名、合計記録値、合計スコア、記録件数、平均きつさを表示する
- 時間式ログの履歴集約では、`value` ではなく `elapsedSeconds` を合計して表示する
- 同じ日に同じ種目を複数回記録した場合は、種目単位でまとめる
- 同じ内容の連続保存を防ぐため、保存後2秒間は保存ボタンを無効化する
- 保存成功後はリザルトオーバーレイを表示する

### 10. トレーナーコメント分岐

- 筋トレ開始時、一時停止時、再開時、種目ログ保存時、今日の目標達成時、休憩チケット自動使用時、入力エラー時にコメントを変更する
- コメントは日本語で、やさしく応援する雰囲気にする
- セッション画面内にもトレーナーコメントを表示し、筋トレ中に見やすくする

### 11. 開発用デバッグ

- 設定画面に開発用デバッグパネルを表示する
- 設定画面に開発者用のスコア調整パネルを表示する
- スコア調整パネルでは、今日の目標スコア、きつさ倍率、評価プロファイル倍率、種目ごとの時間係数・回数係数を編集できる
- スコア調整パネルの変更値は `trelog_dev_scoring_config` に保存する
- 「デフォルトに戻す」ボタンで `trelog_dev_scoring_config` を削除できる
- スコア調整パネルにはテスト計算エリアを表示し、種目、方式、値、きつさ、評価プロファイルからスコアと計算式を確認できる
- 現在の開発者用スコア設定をJSONとして表示し、コピーできる
- `trelog_dev_scoring_config` が有効な場合は「開発者用スコア設定が有効です」と表示する
- 設定画面内に折りたたみ式の「開発用デバッグ」パネルを表示する
- 通常利用UIとは区別できる見た目にする
- `DEBUG_MODE` 定数で表示/非表示を切り替える
- 記録データ削除、状態データ削除、全データ削除ができる
- 休憩チケットを+1、0枚に変更できる
- 任意の日付、昨日、3日前のテスト記録を追加できる
- `trelog_records` と `trelog_state` の内容を画面に表示できる

### 12. PWA土台

- `manifest.json` を用意し、アプリ名、説明、開始URL、表示モード、テーマカラー、アイコン参照を定義する
- `service-worker.js` を用意し、α版の主要ファイルをキャッシュできるようにする
- キャッシュ対象は `index.html`、`style.css`、`app.js`、`manifest.json` とする
- 仮アイコンとして `assets/icons/icon.svg` を用意し、manifestから存在するアイコンだけを参照する
- アイコンファイルが存在する場合は、service workerでキャッシュ対象に含める
- `index.html` からmanifestを読み込み、theme-colorを指定する
- `index.html` でservice workerを登録する
- α版ではPush通知、スマホウィジェット、クラウド保存、ネイティブアプリ化は実装しない

### 13. セッションオーバーレイ

- 筋トレ開始後、通常画面の上にセッション用オーバーレイを表示する
- セッション画面にはトレーナー画像、トレーナーコメント、現在の種目名、記録方式、経過時間、今日の現在スコア / 目標スコア、目標到達までの目安を表示する
- セッション画面には「一時停止 / 再開」「この種目を記録して終了」「キャンセル」ボタンを表示する
- 「この種目を記録して終了」は既存の種目ログ保存処理を使い、保存後にタイマーをリセットして `idle` に戻る
- 「キャンセル」は保存せずにタイマーをリセットして `idle` に戻る
- 時間式の場合は、セッションの経過秒数を保存値の基準として使う
- 回数式の場合は、入力済みの回数を記録値として使い、セッションの経過時間は履歴の補助情報として残す
- 時間式の見込みスコアは、経過秒数を分換算した値 × 種目時間係数 × きつさ補正で計算する
- 回数式の見込みスコアは、セッション画面内の回数 × 種目回数係数 × きつさ補正で計算する
- 回数式の場合、セッション画面内に回数入力欄と `+1`、`+5`、`+10`、`クリア` ボタンを表示する
- 回数式でセッション画面内の回数が1以上でない場合は保存できない
- 時間式で経過秒数が0秒の場合は保存できない

### 14. リザルトオーバーレイ

- 種目ログ保存成功後、通常画面にすぐ戻らずリザルトオーバーレイを表示する
- リザルト画面は、大見出し、トレーナー画像、トレーナーコメント、今回の記録、獲得報酬、今日の進捗、成長、OKボタンの順に表示する
- 大見出しは通常記録なら「ナイス記録！」、今日の目標達成時なら「今日の目標達成！」、レベルアップ時なら「レベルアップ！」とする
- 今回の記録には、種目、記録値、きつさを表示する
- 獲得報酬には、スコアとEXPを表示する
- 今日の進捗には、今日のスコア / 目標スコア、達成率を表示する
- 成長には、現在レベル、次のレベルまでのEXPを表示する
- リザルト画面はOKボタンで閉じる
- 保存後に今日のスコアが目標スコア以上の場合、「今日の目標達成！」と達成コメントを表示する
- 今日の目標達成済み日付は `claimedGoalRewardDates` で管理し、二重に報酬扱いしない
- 保存前レベルと保存後レベルを比較し、レベルアップした場合は `Lv.X にアップ！` を表示する
- レベル報酬で休憩チケットを獲得した場合は、獲得枚数をリザルト画面に表示する
- 今日の目標達成、レベルアップ、休憩チケット獲得は、該当時だけバッジ風表示にする
- トレーナーコメントは、通常記録、今日の目標達成、レベルアップ、休憩チケット獲得、低きつさ、高きつさで分岐する
- 祝福演出はCSSアニメーションで表示し、今日の目標達成時は紙吹雪多め、レベルアップ時も紙吹雪、通常記録時は控えめなキラキラにする

## 簡易スコア仕様

- 時間式: `elapsedSeconds / 60` × 種目係数 × きつさ補正 × 評価レベル補正
- 回数式: 回数 × 種目係数 × きつさ補正 × 評価レベル補正
- 最後に評価レベル補正をかける
- きつさ補正は、1 = 0.8、2 = 0.9、3 = 1.0、4 = 1.1、5 = 1.2
- 評価レベル補正は、初心者 = 1.2、標準 = 1.0、慣れている = 0.85、高負荷向け = 0.7
- 種目係数は以下とする
- 腕立て: 時間10、回数2、時間/回数対応
- 腹筋: 時間8、回数1.5、時間/回数対応
- スクワット: 時間15、回数1.5、時間/回数対応
- プランク: 時間12、回数0、時間のみ対応
- ストレッチ: 時間3、回数0、時間のみ対応
- 自由種目: 時間5、回数1、時間/回数対応

## localStorage保存形式

- 保存キーは `trelog_records` とする
- 1回の保存は以下の形式の1つの種目ログとして保存する
- `id`: string
- `date`: string
- `exerciseId`: string
- `exerciseName`: string
- `mode`: `"time"` または `"reps"`
- `value`: number。時間式では互換・表示用の分換算値
- `unit`: `"分"` または `"回"`
- `intensity`: number
- `elapsedSeconds`: number。時間式では1秒以上の経過秒数を保存する
- `score`: number
- `exp`: number
- `musicFileName`: string
- `createdAt`: string

## localStorage状態保存形式

- 保存キーは `trelog_state` とする
- 初回起動時は以下の値で初期化する
- `restTickets`: `2`
- `restDates`: `[]`
- `claimedGoalRewardDates`: `[]`
- `claimedLevelRewards`: `[]`
- `evaluationProfile`: `"standard"`
- 旧形式の `freezeTickets` と `frozenDates` が残っている場合は、`restTickets` と `restDates` に移行する
- 既存状態に `evaluationProfile` がない場合は `"standard"` で補完する

## 非機能要件

- スマホ縦画面で見やすい
- 初心者でも理解しやすいコードにする
- データはlocalStorageに保存する
- α版では外部APIを使わない
