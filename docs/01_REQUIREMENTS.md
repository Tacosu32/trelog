# 01_REQUIREMENTS

## α版の機能要件

### 1. 日付・記録

- 今日の日付を表示する
- 1回の保存は、1つの種目ログとして扱う
- 1日に複数の種目ログを保存できる
- 日ごとの達成判定は、その日に1件以上ログがあれば達成扱いとする
- 過去の履歴を表示する
- 保存ボタンを押すと、その時点で1つの種目ログとして履歴に追加する
- 保存後はタイマーをリセットし、開始前の状態に戻す
- 種目ログはlocalStorageの `trelog_records` に保存する
- 起動時にlocalStorageから記録を読み込み、履歴表示を復元する

### 2. 筋トレ入力

- 種目を1つ選択して記録できる
- 種目プルダウンには、腕立て、腹筋、スクワット、プランク、ストレッチ、その他 / 新規追加を表示する
- 「その他 / 新規追加」を選んだ場合、新しい種目名を入力できる
- 記録方式は「時間」または「回数」から選択できる
- 時間式の場合は、タイマーの経過時間を記録値として反映できる
- 回数式の場合は回数を入力できる
- 回数式の場合、タイマーの経過時間は補助情報として履歴に表示する
- きつさを1〜5で選択できる
- メモを任意で入力できる
- 「今日のクエスト候補」チェックボックス欄は廃止する

### 3. 今日の目標

- 今日の目標カードを表示する
- 今日の目標カードには、今日の目標スコア、現在スコア、達成率、おすすめ目安を表示する
- 今日の目標スコアは固定で100ptとする
- 今日の日付のログだけを集計して、今日の現在スコアを表示する
- 現在の種目、記録方式、きつさ、現在スコア、今日の目標スコアに応じて、今日の目標到達までの目安を自動更新する
- 目標到達までの必要量は、残りスコア ÷ 種目係数 ÷ きつさ補正で計算し、小数点が出る場合は切り上げる
- 今日のスコアが100pt以上になったら今日の目標達成とする
- 今日の目標達成時、その日付を達成済みとして保存し、達成コメントを表示する
- 今日の目標達成では休憩チケットを付与しない
- 同じ日に今日の目標達成報酬を複数回受け取ることはできない

### 4. タイマー

- 筋トレ開始ボタンを押すとタイマーが開始する
- 実行中はボタン表示を「一時停止」に変更する
- 一時停止中はボタン表示を「再開」に変更する
- 保存後はタイマーをリセットする

### 5. 音楽アップロード

- ユーザーが音楽ファイルを選択できる
- 選択した音楽を再生できる
- 音楽再生と筋トレ開始を同じ画面で行える

### 6. 評価

- 種目、記録方式、入力値、きつさから簡易スコアを計算する
- 今回は簡易スコアをEXPとして保存する
- 累計EXPを表示する
- 全ログのEXPを合計して累計EXPを表示する
- `level = Math.floor(totalExp / 100) + 1` でレベルを表示する
- 現在レベル内のEXP進捗バーを表示する
- 現在レベル、現在EXP、次のレベルまであと何EXPかを表示する
- 5レベルごとに休憩チケットを1枚付与する
- レベル報酬は `claimedLevelRewards` で受け取り済みレベルを管理し、二重受け取りを防ぐ
- 今日の評価コメントを表示する

### 6.1 今週ランク

- 今週ランクを表示する
- 今週は月曜始まりとする
- 今週の記録日数と週間スコアからランクを算出する
- S: 今週5日以上記録、または週間スコア500以上
- A: 今週4日以上記録、または週間スコア350以上
- B: 今週3日以上記録、または週間スコア200以上
- C: 今週1〜2日記録
- D: 今週記録なし
- 可能なら、今週の記録日数と週間スコアも小さく表示する

### 7. 継続

- 連続記録日数を計算する
- その日に1件以上ログがあれば達成日とする
- restDatesに含まれる日を休憩日とする
- 継続日数計算では、記録日と休憩日の両方を継続扱いにする
- 今日に記録または休憩日があれば今日から数える
- 今日に記録がなく、昨日が記録日または休憩日なら昨日から数える
- 連続している日付だけをカウントし、未記録かつ未休憩の日があればそこで終了する
- 起動時に、昨日以前の未記録日が継続を途切れさせる場合、休憩チケットを自動消費して継続を維持する
- 今日に記録がない場合、今日はまだ休憩日にしない
- 複数日空いている場合は、チケットがある分だけ過去の日付を休憩日にしてよい
- チケットが足りない場合は、それ以上は休憩日にしない
- 休憩チケットの枚数を表示する

### 8. トレーナー

- トレーナー画像を表示する
- 状況に応じた定型コメントを表示する
- α版では会話機能は実装しない

### 9. 履歴

- 種目ログごとに履歴を表示する
- 履歴には、日付、種目、記録方式、分または回、きつさ、簡易スコア、EXPを表示する
- 回数式の場合は、タイマーの経過時間を補助情報として表示する
- 同じ内容の連続保存を防ぐため、保存後2秒間は保存ボタンを無効化する

### 10. トレーナーコメント分岐

- 筋トレ開始時、一時停止時、再開時、種目ログ保存時、今日の目標達成時、休憩チケット自動使用時、入力エラー時にコメントを変更する
- コメントは日本語で、やさしく応援する雰囲気にする

### 11. 開発用デバッグ

- 画面下部に折りたたみ式の「開発用デバッグ」パネルを表示する
- 通常利用UIとは区別できる見た目にする
- `DEBUG_MODE` 定数で表示/非表示を切り替える
- 記録データ削除、状態データ削除、全データ削除ができる
- 休憩チケットを+1、0枚に変更できる
- 任意の日付、昨日、3日前のテスト記録を追加できる
- `trelog_records` と `trelog_state` の内容を画面に表示できる

### 12. PWA土台

- `manifest.json` を用意し、アプリ名、説明、開始URL、表示モード、テーマカラー、アイコン参照を定義する
- `service-worker.js` を用意し、α版の主要ファイルをキャッシュできるようにする
- キャッシュ対象は `index.html`、`style.css`、`app.js`、`manifest.json` とする
- 仮アイコンとして `assets/icons/icon.svg` を用意し、manifestから存在するアイコンだけを参照する
- アイコンファイルが存在する場合は、service workerでキャッシュ対象に含める
- `index.html` からmanifestを読み込み、theme-colorを指定する
- `index.html` でservice workerを登録する
- α版ではPush通知、スマホウィジェット、クラウド保存、ネイティブアプリ化は実装しない

## 簡易スコア仕様

- 時間式: 分 × 種目係数 × きつさ補正
- 回数式: 回数 × 種目係数 × きつさ補正
- きつさ補正は、1 = 0.8、2 = 0.9、3 = 1.0、4 = 1.1、5 = 1.2
- 種目係数は以下とする
- 腕立て: 時間10、回数2
- 腹筋: 時間8、回数1.5
- スクワット: 時間8、回数1.5
- プランク: 時間12、回数0
- ストレッチ: 時間3、回数0
- 自由種目: 時間5、回数1

## localStorage保存形式

- 保存キーは `trelog_records` とする
- 1回の保存は以下の形式の1つの種目ログとして保存する
- `id`: string
- `date`: string
- `exerciseId`: string
- `exerciseName`: string
- `mode`: `"time"` または `"reps"`
- `value`: number
- `unit`: `"分"` または `"回"`
- `intensity`: number
- `elapsedSeconds`: number
- `score`: number
- `exp`: number
- `musicFileName`: string
- `createdAt`: string

## localStorage状態保存形式

- 保存キーは `trelog_state` とする
- 初回起動時は以下の値で初期化する
- `restTickets`: `2`
- `restDates`: `[]`
- `claimedGoalRewardDates`: `[]`
- `claimedLevelRewards`: `[]`
- 旧形式の `freezeTickets` と `frozenDates` が残っている場合は、`restTickets` と `restDates` に移行する

## 非機能要件

- スマホ縦画面で見やすい
- 初心者でも理解しやすいコードにする
- データはlocalStorageに保存する
- α版では外部APIを使わない
