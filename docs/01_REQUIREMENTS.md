# 01_REQUIREMENTS

## α版の機能要件

### 1. 日付・記録

- 今日の日付を表示する
- 1回の保存は、1つの種目ログとして扱う
- 1日に複数の種目ログを保存できる
- 日ごとの達成判定は、その日に1件以上ログがあれば達成扱いとする
- 過去の履歴を表示する
- 保存ボタンを押すと、その時点で1つの種目ログとして履歴に追加する
- 保存後はタイマーをリセットし、開始前の状態に戻す

### 2. 筋トレ入力

- 種目を1つ選択して記録できる
- 種目プルダウンには、腕立て、腹筋、スクワット、プランク、ストレッチ、その他 / 新規追加を表示する
- 「その他 / 新規追加」を選んだ場合、新しい種目名を入力できる
- 記録方式は「時間」または「回数」から選択できる
- 時間式の場合は、タイマーの経過時間を記録値として反映できる
- 回数式の場合は回数を入力できる
- 回数式の場合、タイマーの経過時間は補助情報として履歴に表示する
- きつさを1〜5で選択できる
- メモを任意で入力できる
- 「今日のクエスト候補」チェックボックス欄は廃止する

### 3. 今日の目標

- 今日の目標カードを表示する
- 今日の目標カードには、今日の目標スコア、現在スコア、達成率、おすすめ目安を表示する
- 本格的なEXP計算はまだ行わない
- 今日の目標スコア用に簡易スコアを計算する

### 4. タイマー

- 筋トレ開始ボタンを押すとタイマーが開始する
- 実行中はボタン表示を「一時停止」に変更する
- 一時停止中はボタン表示を「再開」に変更する
- 保存後はタイマーをリセットする

### 5. 音楽アップロード

- ユーザーが音楽ファイルを選択できる
- 選択した音楽を再生できる
- 音楽再生と筋トレ開始を同じ画面で行える

### 6. 評価

- 種目、記録方式、入力値、きつさからEXPを計算する
- 累計EXPを表示する
- EXPに応じてレベルを表示する
- 今日の評価コメントを表示する
- α版の今回範囲ではEXP計算は実装せず、今日の目標用の簡易スコアのみ実装する

### 7. 継続

- 連続記録日数を計算する
- 未記録日があった場合、凍結チケットを消費して継続を維持する
- 凍結チケットの枚数を表示する

### 8. トレーナー

- トレーナー画像を表示する
- 状況に応じた定型コメントを表示する
- α版では会話機能は実装しない

### 9. 履歴

- 種目ログごとに履歴を表示する
- 履歴には、日付、種目、記録方式、分または回、きつさ、簡易スコアを表示する
- 回数式の場合は、タイマーの経過時間を補助情報として表示する
- 同じ内容の連続保存を防ぐため、保存後2秒間は保存ボタンを無効化する

## 簡易スコア仕様

- 時間式: 分 × 種目係数 × きつさ補正
- 回数式: 回数 × 種目係数 × きつさ補正
- きつさ補正は、1 = 0.8、2 = 0.9、3 = 1.0、4 = 1.1、5 = 1.2
- 種目係数は以下とする
- 腕立て: 時間10、回数2
- 腹筋: 時間8、回数1.5
- スクワット: 時間8、回数1.5
- プランク: 時間12、回数0
- ストレッチ: 時間3、回数0
- 自由種目: 時間5、回数1

## 非機能要件

- スマホ縦画面で見やすい
- 初心者でも理解しやすいコードにする
- データはlocalStorageに保存する
- α版では外部APIを使わない
