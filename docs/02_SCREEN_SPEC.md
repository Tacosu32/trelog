# 2026-06-10 追記 トレーナー選択 v1 画像/休憩仕上げ

## 設定画面 / トレーナー選択

- プリセットトレーナーの画像プレビューは、選択中トレーナーの `assets/trainer/public/presets/{trainerId}/default.png` を優先表示する。
- ミナ、レイカ、アカリは、それぞれ `default`、`cheer`、`result`、`rest` の4差分画像を持てる。
- カスタム画像がIndexedDBに保存されている場合は、トレーナーを切り替えても表示画像はカスタム画像を優先する。
- カスタム画像優先時は、画像はカスタム画像、名前・説明・口調・セリフは選択中トレーナーに切り替わることを設定画面で説明する。

## セッション画面

- セッション実行中は選択中トレーナーの `cheer` 画像を使う。
- 一時停止中は選択中トレーナーの `rest` 画像を使い、一時停止コメントも選択中トレーナーの口調で表示する。
- 再開時は固定文言ではなく、選択中トレーナーの再開コメントを一時表示する。
- 低負荷や休憩寄りの状態でも、選択中トレーナーのコメントを使う。

## ホーム画面 / 継続カレンダー

- 休憩チケット使用、休憩チケットで継続を守った日、休憩日として表示される日は、選択中トレーナーの休憩/継続系コメントを使う。
- 直近の休憩チケット使用表示では、固定文言ではなく選択中トレーナーの継続保護コメントを表示する。

## リザルト画面

- EXPゲージ背景は薄いベージュ/グレー系にする。
- EXPゲージ進捗は赤、ピンク、オレンジ系の暖色グラデーションにし、通常リザルト、レベルアップ中、報酬獲得時のいずれでも白塗りに見えないようにする。

## セッション操作

- `idle` で筋トレ開始を押すと、セッションオーバーレイを開いて `running` にする
- `running` で一時停止を押すと、タイマーを止めて `paused` にする
- `paused` で再開を押すと、タイマーを再開して `running` にする
- `running` 中でも記録して終了を押せる
- `running` 中に記録して終了を押した場合は、その瞬間にタイマーを止めてから既存保存処理で種目ログを保存し、オーバーレイを閉じる
- `paused` 中も記録して終了を押せる
- キャンセルを押すと、保存せずにオーバーレイを閉じる
- 時間式では、タイマーの経過秒数を分に変換した値を保存値にする
- 回数式では、セッションオーバーレイ内の回数入力値を保存値にする
- セッション中の目標到達目安は、保存済みスコアとセッション中の見込みスコアを合算して更新する

## トレーナー画像

- 公開用画像は `assets/trainer/public/` のみを公開環境で使う
- `trainer_default.png` はホーム、通常時に使う
- `trainer_cheer.png` はセッション中、応援中に使う
- `trainer_result.png` はリザルト、目標達成、レベルアップ、祝福に使う
- `trainer_rest.png` は一時停止、休憩、休憩チケット使用に使う
- 現在の表示優先順位は、IndexedDBカスタム画像、選択中プリセット画像、既存public画像、CSS仮表示の順とする。
- 画像がない場合は既存のCSS仮トレーナー表示へ戻す

## リザルト操作

- 通常記録時は大見出しに「ナイス記録！」を表示する
- 通常記録時は控えめなキラキラを出す
- 今日の目標達成時は大見出しに「今日の目標達成！」を表示し、リザルトオーバーレイ全体に届く紙吹雪を少し多めに出す
- レベルアップ時は大見出しに「レベルアップ！」を表示し、リザルトオーバーレイ全体に届く紙吹雪を出す
- 休憩チケット獲得時も紙吹雪を出す
- 紙吹雪はリザルトオーバーレイ全体を覆うレイヤーで表示する
- 紙吹雪はCSSアニメーションで2〜3秒程度表示し、横方向だけでなく上部から下部まで縦方向に落ちたあと自然に消える
- 今日の目標達成、Lv.UP、休憩チケット獲得は、該当する場合だけバッジ風に表示する
- 休憩チケットを獲得した場合は「休憩チケットをN枚獲得！」を表示する
- OKボタンを押すとリザルトオーバーレイを閉じ、通常画面に戻る

## スマホ操作

- 下部ナビゲーションは画面下部に固定し、親指で切り替えやすい高さにする
- メインコンテンツが下部ナビゲーションに隠れないようにする
- 回数式の +1 / +5 / +10 / クリアなど連打しやすいボタンは、ダブルタップズームが起きにくいように `touch-action: manipulation` を指定する
- ボタンと入力欄はスマホブラウザでズームを誘発しにくいように、基本の文字サイズを16px以上にする
- ボタンは連打時に文字選択が起きにくいようにする

## 履歴表示

- 履歴画面は、年、月、日、種目の階層で折りたたみ表示する
- 初期表示では最新の年、月、日を展開する
- 年行は `2026年` の形式で表示する
- 月行は `6月` の形式で表示する
- 日付行は `2026/06/01　466 / 100pt　達成　3種目` の形式で表示する
- 日付行の達成判定は、その日の合計スコアが目標スコア以上なら `達成`、未満なら `挑戦中` とする
- 種目行は `スクワット　合計120回　180pt　3件　平均きつさ4.0` の形式で表示する
- 時間式の種目行は合計分、回数式の種目行は合計回数を表示する
- `trelog_records` の保存形式は変更せず、画面表示時だけ集計する

  - レベル変化: `Lv.X → Lv.Y`
  - 記録前EXP
  - 獲得EXP
  - 記録後EXP
  - EXPゲージ
  - ゲージ内の現在レベルEXP表示
  - 次のレベルまでのEXP
- リザルト表示時、EXPゲージは記録前状態から記録後状態へCSS transitionで伸びる。
- レベルアップ時は、ゲージ満タン、次レベル0%、最終EXPの順に段階表示する。
- リザルト上部は従来通りトレーナー画像とコメントを主役にし、EXPゲージは成長セクション内で目立つように配置する。
- OKボタンは演出中でも押せる。

## 紙吹雪演出

- 複数条件が同時に起きても、演出クラスは1つだけ付与し、過剰な演出にしない。

# 2026-06-10 追記 トレーナー選択 v1 仕上げ

## 設定画面 / トレーナー選択

- トレーナー選択では、選択中トレーナーの名前、説明、口調、プレビュー画像を表示する。
- カスタム画像が設定されている場合、トレーナー選択画面に「画像はカスタム画像が優先され、切り替えで名前・説明・口調・セリフが変わる」旨を表示する。

- EXPゲージ背景は薄いベージュ/グレー系にする。

# 2026-06-10 追記 設定画面アコーディオン

## 設定画面

- 設定画面の項目は、基本設定、トレーナー設定、カウンセリング、スコア調整、データ管理、アプリ更新、デバッグの7カテゴリで表示する。
- 各カテゴリは見出しをタップ/クリックして開閉する。閉じたカテゴリでは中身を表示しない。
- 初期表示は基本設定を開き、トレーナー設定、カウンセリング、スコア調整、データ管理、アプリ更新、デバッグは閉じる。
- 基本設定には今日の目標スコアの確認、評価プロファイル、現在ビルドの簡易表示を置く。
- トレーナー設定にはトレーナー選択、プリセット説明、カスタム画像優先中の説明、カスタム画像4差分の管理を置く。
- カウンセリング、スコア調整、データ管理、アプリ更新、デバッグは既存フォームやボタンIDを維持したまま各カテゴリ内に移動する。
- スマホではカテゴリ見出しの高さを確保し、「開く/閉じる」が分かる表示を付ける。
