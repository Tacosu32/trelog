# 02_SCREEN_SPEC

## 2026-06-02 追記: オーバーレイ下部ボタンの安全余白

- セッション/リザルトオーバーレイは `position: fixed; inset: 0;` を維持しつつ、`100svh` と `100dvh` を使ってスマホブラウザの可視領域に合わせる。
- オーバーレイ自体とカード内の両方を縦スクロール可能にし、情報量が多くても下部ボタンまで自然に到達できるようにする。
- セッションの一時停止/再開、記録して終了、キャンセルの下には `safe-area-inset-bottom` と追加固定余白を確保する。
- リザルトのOKボタン下にも同じく `safe-area-inset-bottom` と追加固定余白を確保する。
- 下部ナビとメイン画面の下余白も、下部ブラウザUIと重なりにくいように余裕を持たせる。

## 2026-06-02 追記: スマホ下部セーフエリアと更新UI

- 下部ナビは `env(safe-area-inset-bottom)` を含む高さと余白を持ち、メインコンテンツは下部ナビに隠れない padding-bottom を確保する。
- セッションオーバーレイとリザルトオーバーレイは `100dvh` を使い、カード内をスクロール可能にする。
- セッション画面の一時停止/再開、記録して終了、キャンセルの下に safe area 分の余白を確保する。
- リザルト画面のOKボタン下にも safe area 分の余白を確保する。
- 設定画面にアプリ情報セクションを置き、バージョン表示と「アプリ更新を確認」ボタンを表示する。
- デバッグ画面に「アプリキャッシュを削除して再読み込み」ボタンを表示する。

## 2026-06-01 追記: セッション/リザルトの画像中心UI

- セッションオーバーレイでは、トレーナー画像とセリフを上部の主役として表示する。
- セッション中のトレーナー画像は `object-fit: contain` で表示し、正方形画像や透過PNGでも上下が見切れにくいようにする。
- 種目名、記録方式、タイマーは1つのコンパクトなカードにまとめる。タイマーは読みやすさを残しつつ、トレーナー画像より目立ちすぎないサイズにする。
- セッション詳細エリアは控えめな枠線と背景にし、見込みスコア、目標までの目安、音楽情報などを補助情報として扱う。
- リザルトオーバーレイでも、見出し、トレーナー画像、トレーナーコメントを上部で大きく表示し、報酬・進捗・成長データは下に整理する。
- スマホ幅では画像とセリフを縦並びにし、下部ナビや操作ボタンに隠れない余白を確保する。

## 全体構成

- 1つのHTML内にホーム、記録、履歴、設定の4画面を持つ
- 下部固定ナビゲーションで、ホーム、記録、履歴、設定を切り替える
- 現在選択中の画面はナビゲーション上で強調表示する
- メインコンテンツ下部には、下部ナビゲーションに隠れない余白を確保する

## ホーム画面

- アプリ名と今日の日付を表示する
- 連続日数、Lv、EXP、今週ランク、休憩チケットを表示する
- レベル進捗バーを表示する
- 今日の目標カードには目標スコア、現在スコア、達成率、おすすめ目安を表示する
- トレーナーエリアにはトレーナー画像と通常時のコメントを表示する
- ホーム画面のトレーナー画像は default 状態を使う
- 「記録を開始する」ボタンを表示する
- 「記録を開始する」ボタンを押すと記録画面へ移動する

## 記録画面

- 種目ログ入力では種目、記録方式、分または回、きつさを入力する
- 記録方式は選択中の種目が対応するものだけ選べる
- プランク、ストレッチを選んだ場合は回数を無効化し、時間へ自動切り替えする
- 記録方式の下に、使える方式を説明する補足文を表示する
- 音楽エリアでは音楽ファイル選択と再生を行う
- 「筋トレ開始」ボタンからセッションオーバーレイを開く
- 「種目ログを保存」ボタンから、セッションを使わずに現在入力を保存できる

## 履歴画面

- 保存済みの種目ログを、年・月・日・種目単位で集約して表示する
- 年、月、日を折りたたみ展開できる
- 日ごとの合計スコア、目標達成表示、記録種目数を表示する
- 種目ごとの合計値、合計スコア、記録件数、平均きつさを表示する

## 設定画面

- 評価レベル設定を表示する
- 評価レベルは、初心者、標準、慣れている、高負荷向けから選択できる
- 評価レベルの説明文として、初心者は軽い運動も評価されやすく、高負荷向けは厳しめに評価されることを表示する
- スコア調整セクションを表示する
- スコア調整セクションでは、今日の目標スコア、きつさ倍率、評価プロファイル倍率、種目ごとの時間係数・回数係数を入力できる
- テスト計算エリアでは、種目、記録方式、値、きつさ、評価プロファイルを選び、スコア、使用係数、計算式を表示する
- 現在のスコア設定JSONを表示し、コピーできる
- 開発者用スコア設定が有効な場合は、設定画面に状態表示する
- 開発用デバッグパネルを表示する
- 記録データ削除、状態データ削除、全データ削除を実行できる
- 休憩チケット操作、テスト記録追加、localStorage内容表示を実行できる

## セッションオーバーレイ

- 筋トレ開始後、通常画面の上に全画面に近いオーバーレイとして表示する
- スマホ縦画面で、筋トレ中にトレーナー画像、コメント、タイマー、操作ボタンが見やすい配置にする
- セッション画面では細かいデータより、トレーナーと一緒に筋トレしている体験を優先する
- 表示内容は以下とする
- 大きめのトレーナー画像
- トレーナーコメント
- 大きなタイマー
- 現在の種目名
- 記録して終了ボタン
- 一時停止 / 再開ボタン
- キャンセルボタン
- 回数式の場合の今回の回数入力
- 回数式の場合の +1 / +5 / +10 / クリア補助ボタン
- 詳細エリア内に、今日の見込みスコア、目標到達までの目安、選択中の音楽ファイル名、音楽ファイル選択、音楽の再生 / 停止ボタン、ループ再生ON/OFFを表示する
- running中は cheer 画像、一時停止中は rest 画像を表示する
- セッション中のセリフは、開始、10秒、30秒、60秒、一時停止、再開、回数入力、目標達成見込み、高きつさ、低きつさで切り替える

## セッション操作

- `idle` で筋トレ開始を押すと、セッションオーバーレイを開いて `running` にする
- `running` で一時停止を押すと、タイマーを止めて `paused` にする
- `paused` で再開を押すと、タイマーを再開して `running` にする
- `running` 中でも記録して終了を押せる
- `running` 中に記録して終了を押した場合は、その瞬間にタイマーを止めてから既存保存処理で種目ログを保存し、オーバーレイを閉じる
- `paused` 中も記録して終了を押せる
- キャンセルを押すと、保存せずにオーバーレイを閉じる
- 時間式では、タイマーの経過秒数を分に変換した値を保存値にする
- 回数式では、セッションオーバーレイ内の回数入力値を保存値にする
- セッション中の目標到達目安は、保存済みスコアとセッション中の見込みスコアを合算して更新する

## リザルトオーバーレイ

- 種目ログ保存成功後に表示する
- 表示内容は以下の順番とする
- 大見出し
- トレーナー画像
- トレーナーコメント
- 達成バッジ
- 今回の記録: 種目、記録値、きつさ
- 獲得報酬: スコア、EXP
- 今日の進捗: 今日のスコア / 目標スコア、達成率
- 成長: 現在レベル、次のレベルまでのEXP
- OKボタン
- リザルト、目標達成、レベルアップ、祝福時は result 画像を表示する

## トレーナー画像

- 公開用画像は `assets/trainer/public/` のみを公開環境で使う
- `trainer_default.png` はホーム、通常時に使う
- `trainer_cheer.png` はセッション中、応援中に使う
- `trainer_result.png` はリザルト、目標達成、レベルアップ、祝福に使う
- `trainer_rest.png` は一時停止、休憩、休憩チケット使用に使う
- localhostでは `assets/trainer/local/trainer_private.png` が存在する場合のみ優先表示できる
- 画像がない場合は既存のCSS仮トレーナー表示へ戻す

## リザルト操作

- 通常記録時は大見出しに「ナイス記録！」を表示する
- 通常記録時は控えめなキラキラを出す
- 今日の目標達成時は大見出しに「今日の目標達成！」を表示し、リザルトオーバーレイ全体に届く紙吹雪を少し多めに出す
- レベルアップ時は大見出しに「レベルアップ！」を表示し、リザルトオーバーレイ全体に届く紙吹雪を出す
- 休憩チケット獲得時も紙吹雪を出す
- 紙吹雪はリザルトオーバーレイ全体を覆うレイヤーで表示する
- 紙吹雪はCSSアニメーションで2〜3秒程度表示し、横方向だけでなく上部から下部まで縦方向に落ちたあと自然に消える
- 今日の目標達成、Lv.UP、休憩チケット獲得は、該当する場合だけバッジ風に表示する
- 休憩チケットを獲得した場合は「休憩チケットをN枚獲得！」を表示する
- OKボタンを押すとリザルトオーバーレイを閉じ、通常画面に戻る

## スマホ操作

- 下部ナビゲーションは画面下部に固定し、親指で切り替えやすい高さにする
- メインコンテンツが下部ナビゲーションに隠れないようにする
- 回数式の +1 / +5 / +10 / クリアなど連打しやすいボタンは、ダブルタップズームが起きにくいように `touch-action: manipulation` を指定する
- ボタンと入力欄はスマホブラウザでズームを誘発しにくいように、基本の文字サイズを16px以上にする
- ボタンは連打時に文字選択が起きにくいようにする

## 履歴表示

- 履歴画面は、年、月、日、種目の階層で折りたたみ表示する
- 初期表示では最新の年、月、日を展開する
- 年行は `2026年` の形式で表示する
- 月行は `6月` の形式で表示する
- 日付行は `2026/06/01　466 / 100pt　達成　3種目` の形式で表示する
- 日付行の達成判定は、その日の合計スコアが目標スコア以上なら `達成`、未満なら `挑戦中` とする
- 種目行は `スクワット　合計120回　180pt　3件　平均きつさ4.0` の形式で表示する
- 時間式の種目行は合計分、回数式の種目行は合計回数を表示する
- `trelog_records` の保存形式は変更せず、画面表示時だけ集計する

## 2026-06-01 追記: 設定画面のトレーナー画像設定

- 設定画面に「トレーナー画像設定」セクションを表示する。
- 現在のトレーナー画像プレビュー、画像ファイル選択、適用、削除 / デフォルトに戻すボタンを表示する。
- 注意文として、画像はこの端末のブラウザ内に保存され、GitHubにはアップロードされないこと、サイトデータ削除で消えることを表示する。
- カスタム画像がある場合、ホーム、セッション、リザルト、一時停止表示に共通で反映する。

## 2026-06-01 追記: トレーナー画像設定4差分

- 「トレーナー画像設定」には、通常画像、応援画像、祝福画像、休憩画像の4スロットを表示する。
- 各スロットにはプレビュー、画像ファイル選択、適用、削除 / デフォルトに戻すボタンを置く。
- セクション下部に「すべてデフォルトに戻す」ボタンを置く。
- ホーム画面は通常画像、セッションrunningは応援画像、セッションpausedは休憩画像、リザルトは祝福画像を表示する。
- バックアップ欄では、画像本体をData URLとして含めるためJSONサイズが大きくなる可能性を表示する。

## 2026-06-01 追記: バックアップ種別

- データバックアップ欄に「軽量バックアップをエクスポート」と「完全バックアップをエクスポート」を表示する。
- 軽量バックアップはファイルサイズが小さいが、カスタムトレーナー画像本体は復元できない旨を表示する。
- 完全バックアップは画像本体も含むためファイルサイズが大きくなるが、画像も復元できる旨を表示する。
- インポート操作は共通のJSON読み込みUIを使う。

## 2026-06-01 追記: データバックアップ画面

- 設定画面に「データバックアップ」セクションを表示する。
- JSONバックアップの書き出しボタンと、JSONバックアップの読み込み操作を表示する。
- 書き出し対象は `trelog_records`、`trelog_state`、`trelog_dev_scoring_config` とカスタムトレーナー画像のメタ情報とする。
- 読み込み時は「現在の記録と設定を上書きします。続行しますか？」の確認を出す。
- 画像本体は今回のバックアップ対象外であることを表示する。
